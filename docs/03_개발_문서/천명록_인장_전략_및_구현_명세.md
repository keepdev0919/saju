# 기술 명세: 천명록(天命錄) 신성한 인장 및 풀 카드 노출 전략

## 1. 목적 및 철학 (Objective & Philosophy)
**목표**: "콘텐츠 가치 보호(Gating)"와 "품질 증명을 통한 전환 유도(Showcasing)" 사이의 딜레마 해결.

**핵심 컨셉**: **"관리된 노출(The Managed Reveal)"**
블러(Blur)나 잠금 아이콘으로 단순히 가리는 대신, **전체 카드 시스템**을 선명하게 보여주되 그 위에 전통적인 **붉은 낙관(Sacred Seal)**을 찍어 '미해제 표본'임을 명시함.

---

## 2. 전략적 프레임워크 (Strategic Framing)
- **박물관 미학**: 고가의 유물이나 문서가 전시될 때 찍혀 있는 '열람용 인장'의 권위를 차용.
- **심리적 문턱**: 붉은 도장은 예술적 미감을 해치지 않으면서도, "이것은 아직 내 것이 아니다"라는 시각적 선포 역할을 함.
- **개인화 결핍 유도**: 화려한 일러스트는 보여주되, 사용자의 이름이 비어 있는 것을 확인시켜 '각인' 프로세스(결제)로 유도.

---

## 3. UI/UX 상세 사양 (Specification)

### A. 기록 보관소 그리드 (The Teaser)
- **상태**: 기본적으로 무채색(grayscale) 또는 아주 옅은 블러 처리.
- **인터랙션**: 호버(Hover) 시 채도가 돌아오며 "기록 상세보기" 툴팁 노출. 클릭 시 상세 모달 오픈.

### B. 신성한 인장 상세 모달 (The Preview Modal)
- **카드 UI**: 실제 결과 페이지와 동일한 하이엔드 카드 컴포넌트 노출.
- **천명록(天命錄) 인장**: 
  - **시각**: 붉은 전서체 한자 `天命錄` 낙관.
  - **효과**: `Mix-blend-mode: multiply`, `Opacity: 0.7`.
  - **애니메이션**: `seal-arrival` (모달이 열릴 때 하늘에서 도장이 쾅 찍히는 듯한 스케일 다운 연출).
- **개인화 블록**: 
  - 서명 칸에 `[ 인연의 주인을 기다림 ]` 등의 텍스트 배치.
  - 하단 CTA: **"나의 수호신 각인하기"** 버튼 강조.

## 4. 기술적 아키텍처 (Technical Architecture)

### A. 상태 관리 (React)
```javascript
const [selectedTalisman, setSelectedTalisman] = useState(null);
const [showDetail, setShowDetail] = useState(false);
```

### B. CSS 애니메이션 (index.css)
```css
@keyframes seal-arrival {
  0% { transform: scale(4) rotate(0deg); opacity: 0; filter: blur(10px); }
  100% { transform: scale(1) rotate(-15deg); opacity: 0.7; filter: blur(0.3px); }
}

.sacred-seal {
  mix-blend-mode: multiply; /* 종이에 스며든 듯한 질감 구현 */
  animation: seal-arrival 0.8s cubic-bezier(0.175, 0.885, 0.32, 1.275) forwards;
}
```

### C. 자산 보호 (Security)
- 도감용 이미지는 백엔드에서 리사이징된 저용량 버전으로 서빙하거나, 아예 낙관이 물리적으로 합성된 이미지를 내려주어 클라이언트 사이드 변조 방지.

---

## 5. 기대 효과 (Summary)
1. **신뢰 구축**: 압도적 퀄리티의 카드를 미리 보여줌으로써 품질에 대한 의구심 해소.
2. **브랜딩**: 캡처/공유 시 '천명록' 인장이 박힌 채로 유통되어 자연스러운 마케팅 효과.
3. **결제 명분**: "도장이 찍힌 견본"이 아닌, "내 이름이 새겨진 완결된 수호신"에 대한 소유욕 자극.
