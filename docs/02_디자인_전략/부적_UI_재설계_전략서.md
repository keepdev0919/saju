# 부적 UI 고도화: 인터랙션 및 이미지 보존 전략

## 회의 맥락
- **목표**: 부적의 시각적 퀄리티를 온전히 보존하기 위해 '선정 비책' UI를 재설계함.
- **핵심 갈등**: 기존의 카드 상단 '탭' 인터랙션이 사용자가 방금 구매한 부적 이미지의 시각적 완결성을 해침.
- **제안된 솔루션**: 네비게이션을 카드 경계 외부로 이동시키고, 주변 공간(Peripheral space)을 활용한 '이전/다음' 스타일의 네비게이션을 도입하여 프리미엄하고 신비스러운 분위기를 유지함.

## 디자인 원칙
1.  **자산의 성역화 (Asset Sanctity)**: 부적 이미지 자체는 건드리지 않음. 이미지가 공개된 후에는 그 어떤 UI 요소도 작품 위로 직접 떠다니지 않아야 함.
2.  **서사적 흐름**: "이미지"에서 "선정 비책"으로 전환하는 과정이 성스러운 페이지를 넘기거나 비밀을 더 깊이 들여다보는 듯한 느낌을 주어야 함.
3.  **고스트 네비게이션 (Ghostly Navigation)**: UI 요소(화살표/버튼)는 은은하고 반투명해야 하며, 배경(사용자가 언급한 '어둡고 갈색 톤'의 영역)과 일체화된 느낌을 주어야 함.

## 세부 디자인 컴포넌트

### 1. 주변 네비게이션 (The "Ghost Arrows")
- **배치**: `TalismanCard` 좌측과 우측의 어두운 여백 영역에 위치.
- **스타일**:
    - 얇고 우아한 쉐브론 아이콘 (`>`).
    - 낮은 투명도(30-50%)를 가진 다크 앰버 또는 차콜 그레이 색상.
    - 시선을 끌되 방해하지 않는 부드러운 '숨쉬는' 애니메이션 적용.
- **인터랙션**: 우측 화살표 클릭 시 카드 내용이 "비책" 뷰로 슬라이드되고, 좌측 화살표 클릭 시 다시 "이미지" 뷰로 복귀.

### 2. 전환 애니메이션
- **메커니즘**: `TalismanCard` 내부 뒷면에서 CSS `slide-in` 또는 `cross-fade` 사용.
- **분위기**: 전통 한국 미학에 맞춰 은은한 '안개' 또는 '수묵 번짐' 효과와 함께 전환.

### 3. 헤더 및 인디케이터
- **헤더**: "選定 秘策" 또는 "護符 (Talisman)" 제목을 카드 내부가 아닌 카드 위쪽의 고정된 위치로 이동.
- **진행 도트**: 카드 하단에 유저가 현재 어떤 페이지를 보고 있는지 보여주는 미니멀한 도트 인디케이터 (`. .`) 배치.

## 기술적 구현 계획

### [MODIFY] `TalismanCard.jsx`
- 내부 탭 버튼 제거.
- 외부 제어가 가능하도록 `viewMode` ('image' | 'reason')를 props로 노출.
- 전환을 위해 "비책" 뷰의 내부 패딩 및 레이아웃 최적화.

### [MODIFY] `ResultPage.jsx`
- `TalismanCard`를 주변 화살표를 포함하는 정교한 컨테이너로 감쌈.
- 현재 뷰 모드에 대한 상태 관리 구현.
- 현재 페이지에 따라 화살표 가시성 동기화 (예: 첫 페이지에서는 왼쪽 화살표 숨김).

## 검증 계획
- **미학적 체크**: 부적 이미지 위에 UI 오버레이가 100% 제거되었는지 확인.
- **접근성 체크**: 모바일 기기에서도 화살표가 터치하기 충분한 크기인지 확인.
- **로직 체크**: `reason` 데이터가 존재할 때만 "비책" 탭/화살표가 나타나는지 확인.
