## 🔮 사주 결과 도출 방식 추측

각 서비스의 약관 내용을 바탕으로 사주 결과 도출 방식을 추측해봅니다.

### 1. 운세의신 (㈜스포츠조선)

**추측: 전통적 DB 기반 + 부분 자동화**

**근거**:
- "상담 서비스 제공을 위해 회사측 인력의 수동적 작업을 필요로 하는 서비스"와 "자동 정보 콘텐츠" 구분 (제6조)
- AI 관련 면책 조항 전무
- 전통 미디어 기업(스포츠조선) 운영

**추정 방식**:
```
[사용자 입력] → [만세력 DB 조회] → [사주팔자 계산]
                                         ↓
              [사전 작성된 해석 DB 조회] ← [오행/신살 분석]
                                         ↓
                              [템플릿 기반 결과 조합]
```

- 사전에 작성된 해석 문구 DB에서 조건에 맞는 내용 조합
- 일부 프리미엄 서비스는 전문가 직접 작성 가능
- AI 학습/생성 기능 없음

### 2. 청월당 (로켓AI)

**추측: LLM 기반 AI 생성 서비스**

**근거**:
- 회사명 자체가 "로켓AI" - AI 전문 기업
- 제9조 "AI 신뢰성" 조항에서 명시: "인공지능이 생성한 모든 결과"
- "프롬프트 해킹", "시스템 프롬프트", "사용된 모델" 언급 (제21조)
- "유료/무료 서비스는 서로 다른 알고리즘 및 엔진 사용" (제9조)
- "같은 인공지능을 엔진을 사용하더라도... 세부 내용이 달라질 수 있습니다" (제9조)
- AI 생성물의 마케팅/학습 활용 동의 조항 (제8조)
- 워터마킹, 고유 문장 패턴 적용 (제12조)

**추정 방식**:
```
[사용자 입력] → [만세력 API] → [사주팔자 계산]
                                    ↓
                          [오행/신살 분석 엔진]
                                    ↓
              [시스템 프롬프트] → [LLM (GPT/Claude 등)]
                                    ↓
                          [AI 생성 해석 문장]
                                    ↓
                          [워터마킹 후처리]
```

- **무료 서비스**: 경량 모델 또는 기본 프롬프트
- **유료 서비스**: 고급 모델 (GPT-4 등) + 정교한 프롬프트
- 매번 생성 시 미세하게 다른 결과 (LLM 특성)
- 결과물에 비가시적 워터마킹 적용
