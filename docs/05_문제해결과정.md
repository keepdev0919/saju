# 📄 문제 해결 과정 (Case Study): 보관소 카드 열람 UX 개선

본 문서는 천명록(天命錄) 보관소 프로젝트 진행 중 발생한 사용자 경험(UX) 상의 문제점과 이를 해결하기 위한 의사결정 과정을 기록한 문서입니다.

---

## 1. 발견된 문제점 (The Problem)

### 🧐 현상
*   **보관소(Archive)** 페이지에서 특정 수호신 카드를 클릭하여 상세 미리보기를 띄웠을 때, 이를 닫기 위한 유일한 수단이 우측 상단의 **'X' 버튼**이었음.
*   사용자는 총 60개의 갑자 수호신을 하나씩 탐색하며 여러 번 창을 열고 닫아야 하는 상황.

### ⚠️ UX 위험 요소 (Pain Points)
1.  **높은 피로도**: 모바일 기기를 한 손으로 잡고 조작할 때, 우측 상단은 엄지손가락이 닿기 가장 힘든 '데드 존(Dead Zone)'임. 한 번의 열람마다 손 가동 범위를 무리하게 넓혀야 함.
2.  **조작의 불연속성**: 카드 선택은 화면 중앙에서 이루어지는데, 닫기는 구석에서 이루어짐으로써 시선과 동선이 분산됨.
3.  **반복 작업의 장애**: 60개의 카드를 훑어봐야 하는 '도감' 성격의 페이지에서 이러한 미세한 불편함이 누적되어 사용자의 이탈을 유도할 수 있음.

---

## 2. 해결 전략 도출 (Decision Process)

사용자와의 협업 과정을 통해 다음과 같은 핵심 인사이트를 도출했습니다.

> **"사용자는 한 손(엄지)으로만 기기를 조작할 가능성이 높으며, 추가 기능이 없는 단순 미리보기 단계에서는 굳이 버튼을 찾게 할 이유가 없다."**

### 💡 제안된 솔루션
*   **전역 클릭 닫기 (Anywhere Click)**: 닫기 버튼이라는 특정 지점을 조준하게 하지 말고, 화면 어디든(배경 및 카드 본체) 다시 터치하면 닫히도록 설계.
*   **제스처 인터랙션 추가**: 모바일 사용자에게 익숙한 '아래로 밀기(Swipe Down)' 동작을 추가하여 더 직관적인 탈출 경로 제공.

---

## 3. 최종 구현 내용 (Implementation)

### ✅ 기술적 변경 사항
1.  **Click-to-Close**: 모달 배경(`backdrop`)과 콘텐츠 영역 전체에 `onClick` 핸들러를 부여하여 닫기 로직 연결.
2.  **Swipe Down Gesture**: 
    *   `Intersection Observer`가 아닌 `onTouchStart`, `onTouchEnd` 이벤트를 활용하여 사용자의 수직 스와이프 거리 측정.
    *   사용자가 아래로 50px 이상 드래그 시 즉시 창을 닫도록 구현.
3.  **UI 간소화**: 시각적으로 방해가 되던 X 버튼을 제거하고, 하단에 행동 가이드(Instruction) 텍스트를 배치하여 사용자 학습 지원.

---

## 4. 기대 효과 (Expected Outcome)

*   **한 손 조작성(One-handed Usability)** 극대화.
*   사용자 피로도 감소로 인한 **체류 시간 및 도감 탐색량 증가**.
*   제스처 기반의 현대적인 앱 사용 경험(UX) 제공.
